1 Unified Chat with Mode Toggle

Single chat interface with an explicit mode toggle. When in "Respond" mode, selecting a thread pins the context at the top. When in "New Request" mode, it's a fresh conversation.

Pros
  • Clear about current mode
  • Same chat experience either way
  • Context is always visible
Cons
  • Requires mode switching
  • Still two mental models

2 Single Stream with Inline Reply Buttons

All messages appear in one scrolling stream. Each pending question has a "Reply" button. Clicking it sets the reply context. Typing without selecting anyone starts a new request.

Pros
  • Single unified stream
  • Very intuitive "reply" pattern
  • No mode switching needed
Cons
  • Can get long/cluttered
  • Need to scroll to find questions

3 Chat-First with Floating Inbox

Primary experience is chatting with Luna (the PM). Pending questions appear as a floating inbox panel that can be toggled. Selecting a question injects it into the chat as context.

Pros
  • Feels like one simple chat
  • Inbox is secondary, not equal
  • Clear PM personality
Cons
  • Floating panel adds complexity
  • May miss urgent questions

4 Simple Tab Navigation

Two clear tabs: "New Request" and "Team Questions". Each is a full-page experience. The badge on Team Questions shows pending items. Clean separation without complexity.

Pros
  • Dead simple mental model
  • Badge draws attention
  • Full screen for each
Cons
  • Still two separate modes
  • More similar to current design

5 Everything in One View

Pending questions appear as clickable cards in the main view. One input handles both: typing normally sends a new request, clicking a card first sets the reply context. Most like other AI coding tools, but with pending questions surfaced inline.

Pros
  • Single view, no tabs/modes
  • Questions are visible inline
  • Most like other AI chat tools
Cons
  • Requires learning the pattern
  • Could get cluttered

6 WhatsApp-Style Direct Chat

Maximum simplicity: just a direct chat with Luna. When there are pending questions, a notification banner appears inline. Everything is conversation, no separate views at all.

Pros
  • Instantly familiar UX
  • No learning curve
  • Questions as notifications
Cons
  • Less structured for many questions
  • May not scale well
